## 模块说明

### 1. CLI Interface（命令行接口）

**职责**：提供命令行交互入口

**技术栈**：
- Commander.js - 命令行框架
- Chalk - 终端颜色
- Ora - 加载动画
- cli-table3 - 表格输出

**文件**：
```
src/cli/
├── index.ts           # 主入口，命令注册
└── commands/
    ├── import.ts      # 数据导入命令
    ├── list.ts        # 表列表命令
    ├── ask.ts         # 查询命令
    ├── analyze.ts     # 分析命令
    ├── export.ts      # 导出命令
    └── ui.ts          # Web UI 命令
```

### 2. Web UI Interface（Web 界面）

**职责**：提供图形化交互界面

**技术栈**：
- Express - Web 服务器
- CORS - 跨域支持

**API 端点**：

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/tables` | GET | 获取表列表 |
| `/api/import` | POST | 导入数据 |
| `/api/ask` | POST | 自然语言查询 |
| `/api/query` | POST | 执行 SQL |
| `/api/analyze/:table` | GET | 分析表 |
| `/api/schema/:table` | GET | 获取表结构 |
| `/api/export/:table` | GET | 导出表数据 |
| `/api/tables/:table` | DELETE | 删除表 |

**文件**：
```
src/ui/
├── server.ts          # Express 服务器
└── public/
    └── index.html     # 前端页面
```

### 3. SQL Builder（SQL 生成器）

**职责**：将自然语言转换为 SQL

**核心逻辑**：
```
自然语言问题
    │
    ▼
┌─────────────────┐
│  获取表 Schema  │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 构建 LLM Prompt │
│ • 表结构信息    │
│ • 用户问题      │
│ • 输出要求      │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│   调用 LLM API  │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  解析 SQL 结果  │
│ • 去除 Markdown │
│ • 校验语法      │
└─────────────────┘
```

**Prompt 模板**：

```
你是一个 SQL 专家，负责将自然语言问题转换为 SQL 查询。

## 数据库 Schema

表名: {table_name}
列:
{columns_schema}

## 用户问题
{question}

## 要求
1. 生成标准 SQL (DuckDB 兼容)
2. 使用适当的聚合函数和分组
3. 添加 ORDER BY 和 LIMIT (如适用)
4. 不要使用不存在的列
5. 返回纯 SQL，不要解释

## 输出格式
```sql
SELECT ...
```
```

**文件**：
```
src/core/llm/
├── client.ts          # LLM 客户端封装
├── sql-builder.ts     # SQL 生成器
└── prompts.ts         # Prompt 模板
```

### 4. Chart Generator（图表生成器）

**职责**：生成数据可视化图表

**支持类型**：
- bar - 柱状图
- line - 折线图
- pie - 饼图
- scatter - 散点图

**输出格式**：
- ASCII 图表（终端显示）
- HTML 文件（浏览器查看）
- JSON 配置（ECharts 配置）

**文件**：
```
src/utils/charts.ts    # 图表生成工具
```

### 5. Insight Engine（洞察引擎）

**职责**：自动发现数据中的洞察

**洞察类型**：

| 类型 | 说明 | 示例 |
|------|------|------|
| trend | 趋势分析 | 销售额上升趋势 |
| anomaly | 异常检测 | 高缺失率列 |
| correlation | 相关性分析 | 销量与金额强相关 |
| distribution | 分布特征 | 数据量级分析 |

**分析流程**：
```
1. 数据概览
   ├── 行数、列数
   ├── 时间范围
   └── 列类型分布

2. 统计信息
   ├── 数值列：min, max, mean, median, std
   ├── 文本列：distinct count, null count
   └── 日期列：范围检测

3. 洞察生成
   ├── 缺失值分析
   ├── 异常值检测
   ├── 趋势分析
   └── 相关性分析
```

**文件**：
```
src/core/analyzer/
└── insights.ts        # 洞察生成引擎
```

### 6. Importer（数据导入器）

**职责**：解析和导入数据文件

**支持格式**：
- CSV（已实现）
- Excel（规划中）
- Parquet（规划中）

**导入流程**：
```
文件路径
    │
    ▼
┌─────────────────┐
│  检测文件格式   │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│   解析数据      │
│ • 读取内容      │
│ • 推断类型      │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  存入 DuckDB    │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  建立向量索引   │
│ (LanceDB)       │
└─────────────────┘
```

**文件**：
```
src/core/importer/
└── csv.ts             # CSV 解析器
```

### 7. Storage Layer（存储层）

#### DuckDB

**职责**：结构化数据存储和 SQL 执行

**特点**：
- 嵌入式 OLAP 数据库
- 无需独立服务进程
- 支持复杂 SQL 查询
- 高性能列式存储

**使用场景**：
- 数据表存储
- SQL 查询执行
- 聚合统计计算

#### LanceDB

**职责**：向量存储和语义检索

**特点**：
- 轻量级向量数据库
- 无服务器架构
- 支持多模态数据

**使用场景**：
- 表 Schema 向量化
- 智能表推荐
- 语义检索

**文件**：
```
src/core/engine/
├── duckdb.ts          # DuckDB 封装
└── lancedb.ts         # LanceDB 封装
```

## 技术选型理由

### Node.js + TypeScript

| 优点 | 说明 |
|------|------|
| 跨平台 | 支持 Windows、macOS、Linux |
| 类型安全 | TypeScript 提供编译时类型检查 |
| 生态丰富 | npm 生态，依赖丰富 |
| 开发效率 | 开发迭代快速 |

### DuckDB

| 优点 | 说明 |
|------|------|
| 嵌入式 | 无需独立服务进程，简化部署 |
| OLAP 优化 | 列式存储，分析查询性能高 |
| SQL 标准 | 支持完整 SQL 语法 |
| 零配置 | 开箱即用 |

### LanceDB

| 优点 | 说明 |
|------|------|
| 轻量级 | 无服务器架构，低资源占用 |
| 向量检索 | 支持语义相似度搜索 |
| 多模态 | 支持文本、图像等向量 |
| 开源免费 | 无授权成本 |

### OpenAI Compatible API

| 优点 | 说明 |
|------|------|
| 兼容性 | 支持多家 LLM 提供商 |
| 灵活性 | 可切换不同模型 |
| 本地支持 | 支持 Ollama 等本地模型 |

## 性能考量

### 数据量限制

| 数据量 | 处理时间 | 建议 |
|--------|---------|------|
| < 10万行 | < 1秒 | 正常使用 |
| 10万-100万行 | 1-10秒 | 适合大多数场景 |
| 100万-1000万行 | 10-60秒 | 建议采样分析 |
| > 1000万行 | > 60秒 | 建议数据预处理 |

### 优化建议

1. **导入优化**
   - 批量 INSERT 而非逐行插入
   - 禁用索引，导入后重建

2. **查询优化**
   - 使用 LIMIT 限制结果集
   - 避免 SELECT *
   - 合理使用索引

3. **内存优化**
   - 流式处理大文件
   - 及时释放资源

## 安全考量

### 数据安全

- 数据存储在本地，不上传云端
- API Key 通过环境变量配置，不记录到文件
- 支持离线模式（使用本地 LLM）

### 访问控制

- 仅本地访问（localhost）
- 无远程访问接口
- Web UI 无认证（需自行添加）
