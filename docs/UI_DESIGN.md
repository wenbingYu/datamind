# DataMind Web UI 设计文档

## 1. 设计目标

将 DataMind Web UI 从传统表格式管理界面改造为**对话式交互界面**，类似 ChatGPT / Claude 的聊天体验。

### 核心价值

- **零学习成本** - 像聊天一样自然交互
- **一站式体验** - 上传、分析、问答在一个界面完成
- **上下文连贯** - 对话历史保留，支持追问

## 3. 组件设计

### 3.1 消息气泡 (Message Bubble)

#### 用户消息

```
┌──────────────────────────────────────┐
│  销售额最高的 Top 10 产品是哪些？    │
└──────────────────────────────────────┘
                              👤 景闻
```

**样式**：
- 背景：`#2f2f2f`（深灰）
- 圆角：`18px 18px 4px 18px`
- 靠右对齐
- 最大宽度：`70%`
- 头像：用户首字母或图标

#### AI 回复

```
🤖 DataMind
┌──────────────────────────────────────┐
│  以下是销售额最高的 Top 10 产品：    │
│                                      │
│  | 产品名称      | 销售额     |      │
│  |---------------|------------|      │
│  | iPhone 15 Pro | ¥1,234,567 |      │
│  | MacBook Air   | ¥987,654   |      │
│                                      │
│  📊 [查看图表] [导出]                │
└──────────────────────────────────────┘
```

**样式**：
- 背景：`#3a3a3a`（中灰）
- 圆角：`18px 18px 18px 4px`
- 靠左对齐
- 最大宽度：`80%`
- 支持 Markdown 渲染
- 支持表格、代码高亮

### 3.2 文件上传卡片

```
┌─────────────────────────────────────┐
│  📎 sales.csv                       │
│  ─────────────────────────────────  │
│  1,234 行 × 8 列                    │
│  大小: 128 KB                       │
│  ─────────────────────────────────  │
│  [预览] [分析] [删除 ×]             │

**状态**：
- 上传中：进度条

**功能**：

## 4. 交互流程

### 4.1 上传文件流程

```
用户拖拽文件
    │
    ▼
显示上传进度
    │
    ▼
上传完成，显示文件卡片
    │
    ▼
AI 自动回复："已上传 sales.csv，包含 1,234 行数据。你可以问我关于这份文件的问题。"
```

### 4.2 问答流程

```
用户输入问题
    │
    ▼
显示用户消息气泡
    │
    ▼
显示 AI "正在思考..." 动画
    │
    ▼
调用 /api/chat API
    │
    ▼
流式返回 AI 回复
    │
    ▼
渲染 Markdown + 表格/图表
```

### 4.3 追问流程

```
AI 回复后，用户继续提问
    │
    ▼
自动带上已上传文件上下文
    │
    ▼
保持对话连贯性
```

## 9. 组件库选型

### 方案 A：纯 HTML/CSS/JS（当前）

- **优点**：零依赖，加载快
- **缺点**：开发效率低
- **适用**：轻量级需求

### 方案 B：React + Tailwind CSS（推荐）

- **优点**：组件化，开发效率高
- **缺点**：需要构建工具
- **适用**：长期维护

### 方案 C：Vue 3 + Vite

- **优点**：轻量，渐进式
- **缺点**：生态稍弱于 React
- **适用**：中小型项目

  "message": "销售额最高的产品",

  "history": [
    { "role": "user", "content": "..." },
    { "role": "assistant", "content": "..." }

Response: Stream<SSE>
```
